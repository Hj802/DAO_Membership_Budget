# DAO 회비/예산 집행 화면설계서

## 1. 문서 목적

본 문서는 블록체인 기반 DAO 회비/예산 집행 서비스의 화면 구조와 주요 UI 요소를 정의한다.

`DAO_Membership_Budget_PRD_SRS.md` 내부의 화면설계서는 유지하며, 본 문서는 구현과 디자인 작업에서 별도로 참고하기 위한 독립 화면설계서이다.

## 2. 디자인 방향

### 2.1 디자인 컨셉

본 서비스는 단순 투표 앱이 아니라 돈이 걸린 공동 금고 서비스이다. 따라서 과도하게 화려한 Web3 스타일보다 신뢰감 있는 금융/운영 대시보드 스타일을 우선한다.

### 2.2 UI 톤

- 배경: 밝은 회색 계열
- 주요 패널: 흰색 카드
- 텍스트: 진한 차콜 또는 네이비
- 주요 액션: 파란색
- 집행/완료: 초록색
- 취소/종료/위험: 주황색 또는 빨간색
- 주소, 트랜잭션 해시, 컨트랙트 주소: monospace 서체

### 2.3 핵심 UI 원칙

- 지갑 연결 전에는 DAO 목록과 상세 정보를 보여주지 않는다.
- 지갑 연결 후에는 내가 구성원으로 등록된 DAO만 표시한다.
- 비구성원은 웹 서비스에서 DAO 상세, 제안, 증빙, 예산 상세를 볼 수 없다.
- 컨트랙트 주소와 트랜잭션 해시는 블록 익스플로러 링크로 제공한다.
- 상태는 텍스트만으로 표시하지 않고 색상 배지로 함께 표시한다.
- 돈과 관련된 액션은 항상 확인 가능한 상태, 트랜잭션 대기 상태, 성공/실패 상태를 제공한다.

## 3. 공통 레이아웃

### 3.1 지갑 연결 전

지갑 연결 전에는 서비스 소개와 지갑 연결 버튼만 제공한다.

```text
┌────────────────────────────────────────────────────────────────────┐
│ DAO Budget Vault                                      [지갑 연결]  │
├────────────────────────────────────────────────────────────────────┤
│                                                                    │
│   동아리/팀 회비를 스마트컨트랙트 금고에 보관하고,                 │
│   구성원 투표로 승인된 지출만 자동 집행합니다.                     │
│                                                                    │
│   ┌──────────────┐ ┌──────────────┐ ┌──────────────┐              │
│   │ 회비 금고     │ │ 구성원 투표   │ │ 온체인 집행   │              │
│   │ 개인 계좌가   │ │ 1인 1표로     │ │ 승인 조건을   │              │
│   │ 아닌 컨트랙트 │ │ 지출 승인     │ │ 만족하면 송금 │              │
│   └──────────────┘ └──────────────┘ └──────────────┘              │
│                                                                    │
│   지갑을 연결하면 내가 구성원으로 등록된 DAO만 표시됩니다.          │
│   MVP는 Sepolia 테스트넷 ETH를 사용합니다.                         │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```

### 3.2 지갑 연결 후

지갑 연결 후에는 좌측에 내가 속한 DAO 목록을 고정 표시하고, 상단에는 지갑 상태를 제공한다. 좌측 영역은 기능 메뉴가 아니라 DAO 선택 목록이다.

```text
┌──────────────────────┬─────────────────────────────────────────────┐
│ DAO Budget Vault     │ 화면 제목                         Sepolia   │
│                      │ 설명 문구                         0x12...fe │
│ [DAO 생성]           ├─────────────────────────────────────────────┤
│                      │                                             │
│ 내 DAO               │ DAO별 화면 본문                              │
│ - 블록체인 스터디    │                                             │
│   잔액 1.82 ETH      │ 상단 탭:                                    │
│   [투표 중 2]        │ [대시보드] [제안] [예산 내역] [종료]          │
│                      │                                             │
│ - 졸업작품 팀        │                                             │
│   잔액 2.02 ETH      │                                             │
│   [집행 가능 1]      │                                             │
└──────────────────────┴─────────────────────────────────────────────┘
```

### 3.3 DAO 선택 후 내부 탭

DAO를 선택하면 본문 상단에 해당 DAO의 내부 탭을 표시한다.

```text
┌────────────────────────────────────────────────────────────────────┐
│ 블록체인 스터디      0x91af...6bc2 ↗       상태: 활성              │
├────────────────────────────────────────────────────────────────────┤
│ [대시보드] [제안] [예산 내역] [종료]                               │
├────────────────────────────────────────────────────────────────────┤
│ 선택된 탭의 화면 내용                                               │
└────────────────────────────────────────────────────────────────────┘
```

| 내부 탭 | 이동 화면 |
|---|---|
| 대시보드 | UI-004 DAO 대시보드 |
| 제안 | UI-007 제안 목록 |
| 예산 내역 | UI-009 예산 내역 |
| 종료 | UI-010 DAO 종료 제안 생성 |

## 5. 화면별 설계

## 5.1 UI-000 지갑 연결 전 화면

### 목적

사용자가 서비스를 이해하고 지갑 연결을 시작하게 한다.

### 접근 조건

- 지갑 미연결 사용자

### 주요 요소

- 서비스명
- 지갑 연결 버튼
- 서비스 한 줄 설명
- 핵심 가치 카드
- Sepolia 테스트넷 안내
- 지갑 연결 후 내가 속한 DAO만 표시된다는 안내

### 화면 설계

```text
┌────────────────────────────────────────────────────────────────────┐
│ DAO Budget Vault                                      [지갑 연결]  │
├────────────────────────────────────────────────────────────────────┤
│                                                                    │
│   스마트컨트랙트 금고로 회비를 보관하고                            │
│   구성원 투표로 승인된 지출만 집행합니다.                          │
│                                                                    │
│   [회비 금고]        [구성원 투표]        [온체인 집행]             │
│   개인 계좌 대신      승인 없는 지출       조건 충족 시             │
│   컨트랙트 보관       방지                 자동 송금                │
│                                                                    │
│   지갑 연결 후 내가 속한 DAO만 표시됩니다.                         │
│   비구성원은 DAO 상세, 제안, 증빙, 예산 내역을 볼 수 없습니다.      │
│                                                                    │
│                                              [지갑 연결하기]        │
└────────────────────────────────────────────────────────────────────┘
```

### 상태

| 상태 | 처리 |
|---|---|
| 지갑 미연결 | 지갑 연결 버튼 표시 |
| 지갑 연결 요청 중 | 연결 대기 상태 표시 |
| 연결 실패 | 실패 메시지와 재시도 버튼 표시 |
| Sepolia 아님 | 네트워크 변경 안내 표시 |

### 버튼 동작

| 버튼 | 클릭 시 동작 | 이동 대상 |
|---|---|---|
| 지갑 연결 | 브라우저 지갑 연결 요청을 실행한다. 연결 성공 후 Sepolia 네트워크인지 확인한다. | 정상 연결 시 UI-001 내 DAO 목록 |
| 지갑 연결하기 | 상단의 지갑 연결 버튼과 동일하게 동작한다. | 정상 연결 시 UI-001 내 DAO 목록 |

### 예외 처리

- 사용자가 지갑 연결을 거부하면 실패 메시지와 재시도 버튼을 표시한다.
- 연결된 네트워크가 Sepolia가 아니면 네트워크 변경 안내를 표시한다.
- 지갑 연결 전에는 DAO 목록, DAO 상세, 제안, 증빙, 예산 내역을 표시하지 않는다.

## 5.2 UI-001 내 DAO 목록

### 목적

연결된 지갑이 구성원으로 등록된 DAO만 표시한다.

### 접근 조건

- 지갑 연결 사용자

### 주요 요소

- 활성 DAO / 종료 DAO 필터
- DAO 생성 버튼
- 좌측 DAO 목록
- 본문 DAO 목록 또는 빈 상태
- 잔액, 구성원 수, 진행 중 제안 수
- DAO 금고 컨트랙트 주소

### 화면 설계

```text
┌──────────────────────┬─────────────────────────────────────────────┐
│ DAO Budget Vault     │ 내 DAO 목록                  [동기화] [생성] │
│ [DAO 생성]           ├─────────────────────────────────────────────┤
│                      │ [활성 DAO] [종료 DAO]                        │
│ 내 DAO               │                                             │
│ > 블록체인 스터디    │ ┌─────────────────────────────────────────┐ │
│   잔액 1.82 ETH      │ │ 블록체인 스터디      [투표 중 2] [열기] │ │
│   투표 중 2          │ │ 0x91af...6bc2 ↗                         │ │
│                      │ │ 잔액 1.82 ETH  구성원 12명              │ │
│ - 졸업작품 팀        │ └─────────────────────────────────────────┘ │
│   잔액 2.02 ETH      │                                             │
│   집행 가능 1        │ ┌─────────────────────────────────────────┐ │
│                      │ │ 졸업작품 팀        [집행 가능 1] [열기] │ │
│ 종료 DAO             │ │ 0x77cd...4a10 ↗                         │ │
│ - 2025 행사팀        │ │ 잔액 2.02 ETH  구성원 5명               │ │
│                      │ └─────────────────────────────────────────┘ │
└──────────────────────┴─────────────────────────────────────────────┘
```

### 빈 상태

```text
┌────────────────────────────────────────────────────────────────────┐
│ 아직 구성원으로 등록된 DAO가 없습니다.                             │
│ DAO를 새로 만들거나, 다른 사용자가 내 지갑 주소를 구성원으로        │
│ 등록하면 이곳에 표시됩니다.                                        │
│                                                    [DAO 생성]       │
└────────────────────────────────────────────────────────────────────┘
```

### 버튼/클릭 동작

| 버튼/영역 | 클릭 시 동작 | 이동 대상 |
|---|---|---|
| 좌측 DAO 항목 | 선택한 DAO를 현재 작업 DAO로 설정하고 본문을 해당 DAO의 대시보드로 바꾼다. | UI-004 DAO 대시보드 |
| 본문 DAO 카드의 열기 | 선택한 DAO를 현재 작업 DAO로 설정하고 대시보드로 이동한다. | UI-004 DAO 대시보드 |
| 활성 DAO | 좌측 목록과 본문 목록에서 활성 DAO만 표시한다. 종료 DAO는 숨긴다. | UI-001 내 DAO 목록 유지 |
| 종료 DAO | 좌측 목록과 본문 목록에서 종료 DAO만 표시한다. 종료 DAO에서는 입금, 제안 생성, 투표, 집행 액션을 제공하지 않는다. | UI-001 내 DAO 목록 유지 |
| 동기화 | Factory 이벤트와 DAO 이벤트를 기준으로 내 DAO 목록을 재조회한다. 선택된 DAO가 목록에 남아 있으면 선택 상태를 유지한다. | UI-001 내 DAO 목록 유지 |
| DAO 생성 | DAO 생성 입력 화면으로 이동한다. | UI-003 DAO 생성 |

### 좌측 DAO 목록 동작

- 지갑 연결 직후에는 좌측 DAO 목록에 내가 구성원으로 등록된 DAO만 표시한다.
- 좌측 DAO 항목을 클릭하면 본문은 항상 해당 DAO의 UI-004 대시보드로 이동한다.
- 현재 선택된 DAO는 좌측 목록에서 강조 표시한다.
- 선택된 DAO가 종료 상태이면 대시보드에는 읽기 전용 정보와 예산 내역 진입만 제공한다.
- 내가 속한 DAO가 없으면 좌측 목록에는 빈 상태와 `DAO 생성` 버튼만 표시한다.

## 5.3 UI-002 지갑 연결 상태

### 목적

연결 지갑, 네트워크, 권한 상태를 표시한다.

```text

│ 네트워크       Sepolia Testnet                                     │
│ 권한           DAO 구성원                                          │
│                                                                    │
│ [지갑 새로고침] [네트워크 변경]                                    │

```

### 버튼 동작

| 버튼 | 클릭 시 동작 | 이동 대상 |
|---|---|---|
| 복사 | 전체 지갑 주소를 클립보드에 복사한다. | UI-002 유지 |

| 네트워크 변경 | Sepolia 네트워크 전환 요청을 지갑에 보낸다. | 성공 시 UI-001 내 DAO 목록 또는 이전 화면 |

### 예외 처리

- 네트워크 변경을 사용자가 거부하면 안내 메시지를 표시한다.
- 지갑 연결이 끊기면 UI-000 지갑 연결 전 화면으로 이동한다.

## 5.4 UI-003 DAO 생성

### 목적

조직명, 추가 구성원 주소, 기본 승인 기준을 입력해 DAO를 생성한다.

### 주요 정책

- 생성자는 자동 구성원에 포함된다.
- 입력하는 구성원 주소는 생성자를 제외한 추가 구성원 주소이다.
- 생성자 포함 전체 구성원 수는 최대 20명이다.

### 화면 설계

```text
┌────────────────────────────────────────────────────────────────────┐
│ DAO 생성                                                           │
├────────────────────────────────────────────────────────────────────┤
│ 조직명                                                             │
│ [ 블록체인 스터디                                             ]    │
│                                                                    │
│ 추가 구성원 지갑 주소                                               │
│ ┌────────────────────────────────────────────────────────────────┐ │
│ │ 0x91af...6bc2                                                   │ │
│ │ 0x77cd...4a10                                                   │ │
│ │ 0x1a2b...e4d2                                                   │ │
│ └────────────────────────────────────────────────────────────────┘ │
│                                                                    │
│ 기본 승인 기준                                                     │
│ (●) 과반 찬성       ( ) 2/3 찬성                                   │
│                                                                    │
│ 생성자 자동 포함: 0x12ab...91fe                                    │
│ 구성원 수: 4명 / 최대 20명                                         │
│                                                                    │
│                                           [취소] [DAO 생성]         │
└────────────────────────────────────────────────────────────────────┘
```

### 버튼 동작

| 버튼 | 클릭 시 동작 | 이동 대상 |
|---|---|---|

### 버튼 검증

- 조직명이 비어 있으면 생성하지 않는다.
- 추가 구성원 주소 형식이 잘못되면 생성하지 않는다.
- 중복 주소가 있으면 생성하지 않는다.
- 생성자 포함 전체 구성원이 20명을 초과하면 생성하지 않는다.
- 트랜잭션 전송 후에는 pending 상태를 표시한다.
- 성공하면 `DAOCreated` 이벤트 동기화 후 화면을 갱신한다.

### 버튼 예외

- 이미 투표한 구성원에게는 찬성/반대 버튼을 비활성화한다.
- `block.timestamp >= deadline`이면 찬성/반대 버튼을 숨긴다.
- 취소됨, 부결, 집행 실패 상태에서는 투표/집행/증빙 등록 버튼을 숨긴다.
- 집행 실패 상태는 재집행을 허용하지 않는다.

## 5.11 UI-010 DAO 종료 제안 생성

### 목적

DAO 종료 사유를 입력하고 잔액 균등 반환을 위한 종료 제안을 생성한다.

### 접근 조건

- DAO 구성원
- 진행 중 제안 없음
- 집행 가능 제안 없음

### 화면 설계

```text
┌────────────────────────────────────────────────────────────────────┐
│ DAO 종료 제안 생성                                  [생성 가능]     │
├────────────────────────────────────────────────────────────────────┤
│ 현재 잔액: 1.82 ETH                                                │
│ 반환 대상: 12명                                                    │
│ 1인 예상 반환액: 0.151666 ETH                                      │
│ 나머지 wei 수신자: DAO 생성자                                      │
│                                                                    │
│ 종료 제안 사유                                                     │
│ ┌────────────────────────────────────────────────────────────────┐ │
│ │ 스터디 종료로 남은 회비를 구성원에게 균등 반환합니다.           │ │
│ └────────────────────────────────────────────────────────────────┘ │
│                                                                    │
│ 종료 후에는 입금, 지출 제안 생성, 투표, 집행을 할 수 없습니다.     │
│                                                                    │
│                                      [취소] [종료 제안 생성]        │
└────────────────────────────────────────────────────────────────────┘
```

### 버튼 동작

| 버튼 | 클릭 시 동작 | 이동 대상 |
|---|---|---|
| 종료 제안 생성 | 종료 가능 조건을 다시 확인한 뒤, 종료 사유를 DB에 저장하고 종료 제안 `createProposal` 트랜잭션을 요청한다. 성공 시 DAO 상태를 종료 투표 중으로 갱신한다. | 성공 시 UI-008 종료 제안 상세/투표 |
| 취소 | 현재 선택된 DAO의 대시보드로 돌아간다. | UI-004 DAO 대시보드 |

### 버튼 검증

- 진행 중 제안이 있으면 생성하지 않는다.
- 집행 가능 제안이 있으면 생성하지 않는다.
- 종료 제안 사유가 비어 있으면 생성하지 않는다.
- 종료 제안의 온체인 값은 `proposalType = 1`, `amountWei = 0`, `recipient = address(0)`으로 요청한다.
- 종료 투표 중에는 입금, 신규 지출 제안 생성, 일반 제안 집행 버튼을 비활성화한다.
